Built for the field

Everything you'd run on a desktop NINA install, packaged for headless machines and a Web UI that loads in 50 ms on a tethered phone.

⚙️

9 device types

Camera, mount, focuser, filter wheel, rotator, dome, flat panel, weather, guider, over INDI or ASCOM/Alpaca, with auto-discovery on the LAN.

🎯

Plate-solve & Slew-and-Center

ASTAP, PlateSolve3, Astrometry.net (online + local), primary + blind fallback. Auto-pushes the solved focal length back into the rig.

📡

PHD2 guiding, fully managed

JSON-RPC client with live RA/Dec chart, dither, settle params, profile switcher. Auto-launches PHD2 at startup if you opt in.

🌲

Advanced Sequencer (tree)

Containers, loop conditions, event triggers (auto-focus on temp/HFR, meridian flip, dither, safety abort). Drag-drop tree editor in the browser.

🧩

Mosaic planner

N×M grid with cos(δ) correction + serpentine slew order. One click lowers the plan into a runnable sequence.

📸

Live stack & bandwidth-adaptive stream

EAA-style live stacking over WebSocket with LZ4 compression. Auto-downgrades raw → JPEG when WiFi gets flaky.

🌐

Remote relay (no port-forward)

Outbound WebSocket to a tiny relay server gives you internet access without DDNS. Per-tenant quotas, mTLS, web admin, built-in Let's Encrypt.

🔌

Plugin system

Drop a .NET .dll into plugins/, ship custom sequencer instructions. Isolated load context, bad plugins can't take the host down.

🛰️

Offline sky viewer

Canvas-based starfield with 150 brightest stars + constellations + DSO catalog. Works without internet, perfect for dark-site observatories.

Get started in 30 seconds

One static binary, no runtime install. Pick your platform.

RPi

Raspberry Pi 4 / 5

git clone https://github.com/DanWBR/NINA.Polaris.git
cd NINA.Polaris
./deploy/publish-linux-arm64.sh
./publish/linux-arm64/NINA.Polaris

Then point any browser at http://nina-<hostname>.local:5000. mDNS announcer ships built-in.

Windows

Windows mini PC

git clone https://github.com/DanWBR/NINA.Polaris.git
cd NINA.Polaris
.\deploy\publish-win-x64.ps1
.\publish\win-x64\NINA.Polaris.exe

Or -InstallService for a Windows Service that survives reboots. Discovers ASCOM Remote (Alpaca) drivers on the LAN automatically.

Docker

Container

docker run -d --network host \
  -v $(pwd)/config:/config \
  -v $(pwd)/images:/images \
  ghcr.io/danwbr/nina-polaris:latest

Multi-arch image (arm64 + amd64). Compose file in the repo includes indiserver in the same stack.

Built on the same stuff you'd choose yourself

  • ASP.NET Core 10 · Minimal API, single-binary publish, runs on Linux ARM64 / x64 / Windows
  • Alpine.js + plain JS · the UI is ~3700 lines, no build step, hot-editable
  • WebSockets · binary image relay (LZ4 frames) + status broadcast at 1 Hz
  • INDI over TCP/XML + ASCOM Alpaca over HTTP, auto-discovery for both
  • Chart.js for guiding / focus / HFR / temperature plots
  • WebGL2 shader pipeline for client-side debayer + MTF stretch (with CPU fallback)
  • LettuceEncrypt for the relay server's built-in Let's Encrypt
  • Zero cloud dependencies, every byte of catalog, FITS writing, plate-solve dispatch, sequencer engine lives in the binary
